Codex 配置

仅作用于 codex CLI 走的 Responses->Chat 转换路径（Claude Code 走 Anthropic 直转不受影响）。

透传思考强度参数（reasoning_effort）
开启后，调用方传来的思考强度（如 codex 的 reasoning_effort=high）作为 reasoning_effort 原值透传给上游；关闭则保守跳过（不发该字段）。兼容性参考：GLM/DeepSeek 原生认此字段；MiniMax 接受但不调深度；火山/讯飞/Kimi 参数名不同，传过去多半被忽略（不报错）。默认开启，若实测出现 400 再关闭。值 low/medium/high 不做翻译。
思考过程注入：<think> 标签
将 codex 回传的历史 reasoning 以 <think>...</think> 标签注入 assistant 消息 content。适用：MiniMax-M3（Interleaved Thinking，多轮工具调用必需）。默认关闭。
思考过程注入：reasoning_content 字段
将 codex 回传的历史 reasoning 以独立 reasoning_content 字段注入 assistant 消息。适用：DeepSeek/GLM/Kimi 思考模式原生字段（多轮工具调用必需，缺失会 400）。默认开启。
模型原生支持 Responses 协议（透传，跳过 Responses↔Chat 转换）
开启后，/openai 端点对该 mapping 不做 Responses↔Chat 双向转换，直接按 provider.openai_url 派生 /responses 端点原样转发（仿 Anthropic handler 直转语义）。适用上游：原生支持 Responses API 的 OpenAI 兼容端点；其余上游（仅暴露 /chat/completions）开启后会被 fallback 兜底。开启时，上方两个"思考过程注入"开关自动关闭、三方互斥。默认关闭。
