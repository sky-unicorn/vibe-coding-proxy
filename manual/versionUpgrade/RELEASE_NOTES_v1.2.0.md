# 发布说明（复制到 Describe this release 正文）

## 🎉 Vibe Coding 服务转发 v1.2.0

基于 Flask 的多供应商 AI API 代理服务，把多家上游 LLM 统一抽象成 **Anthropic Messages API**、**OpenAI Chat Completions API**、**OpenAI Responses API** 三种接口对外暴露，并配套可视化 Web 管理界面。本版本延续 **Windows 免安装 exe** 双击即运行的形态，重点增强了计费灵活性、数据库结构演进的安全保障，并修复了 max_tokens 转发的潜在 400 错误。

### ✨ 本版本更新（v1.1.1 → v1.2.0）

- **余额计费新增「缓存命中价格」配置**
  - 此前余额（balance）计费模式下，缓存命中 token（`cache_read`）按输入价格的 10% 硬编码计费，无法适配不同供应商的实际缓存折扣
  - 计费设置抽屉新增「缓存命中价格（/百万 Token）」输入框；余额扣减改用此配置值
  - **留空（或 0）时自动回退为输入价格的 10%**，与 v1.1.1 行为完全兼容，无需改动存量配置

- **`max_tokens` 转发改为按模型配置上限钳制**（修复潜在 400 错误）
  - 此前模型映射配置的 `max_tokens` 仅在客户端未传时作为默认值填充；当客户端传入超过模型上限的值（例如客户端传 64000，而模型上限 32768），会被原样转发，导致上游 provider 直接返回 400
  - 现统一为「上限钳制 + 填空」：配置了 `max_tokens` 时，客户端未传或 ≤0 则填配置值，客户端传入超过配置值则裁剪到配置值，确保配置上限真正生效
  - 三条转发路径（OpenAI 直转、Anthropic 转发、Responses / 流式降级）已全部同步

- **引入数据库版本号迁移框架**（内部可靠性增强）
  - 采用 `PRAGMA user_version` 版本号线性迁移机制，替换原先依赖手写函数与列存在性检测的零散迁移逻辑
  - 新增 `_MIGRATIONS` 注册表 + `run_migrations` 按序前向执行，每步递增版本号、崩溃可断点续跑
  - **启动自检兜底**：派生期望 schema 做反射对比，遗漏迁移会直接启动失败，把「忘了写迁移」从静默故障变为启动即暴露
  - 老库首次启动会自动基线校准到当前版本并跑必要迁移，不重跑、不丢数据；全新库直接获得最新结构

- **供应商表单 API Key 输入框支持显示 / 隐藏切换**
  - API Key 输入框新增眼睛图标切换按钮，方便输入或核对密钥时临时查看明文
  - 默认隐藏；打开与关闭供应商弹窗时自动重置为隐藏状态，确保密钥默认不可见

### 🚀 快速开始

1. 下载 `VibeCodingProxy-v1.2.0.exe`，放到任意目录（建议新建一个空文件夹，数据库文件会生成在同目录）。
2. 如需自定义端口或管理员，在同目录新建 `proxy.ini`：

    ```ini
    [server]
    port = 5000

    [admin]
    username = admin
    password = admin123
    ```

3. 双击运行，弹出的控制台窗口会显示各端点地址。
4. 浏览器打开 **http://localhost:5000**（端口随 `proxy.ini` 变化），首次启动会自动初始化数据库并创建管理员账号。
5. 在 Web 界面配置上游提供商、模型映射、API Key 即可开始转发。

### ⚠️ 升级说明

- **从 v1.0.0 / v1.1.1 的 exe 升级**：把旧版本同目录的 `proxy.db` 复制到新 exe 同目录，即可保留历史数据。
- 首次启动 v1.2.0 时，新版本迁移框架会自动校准库版本并补齐「缓存命中价格」列（`cache_read_price_per_million`），按备份 → 重建 → 回填的流程执行，历史计费配置不丢失；该列默认为 0，计费行为与旧版一致。
- `proxy.ini` 是可选文件；管理员账号 / 密码以该文件为准，修改后重启会覆盖库中已有管理员，日常改密请直接编辑该文件。
